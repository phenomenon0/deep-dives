Stack vs Heap: The Escape Analysis Game

Go's compiler decides where to put your buffer. If a value "escapes" the function — gets returned, stored in a struct, passed to an interface — it goes on the heap. Otherwise, stack.

// STACK: compiler sees this stays local. Zero GC cost.
func encodeHeader() []byte {
    var buf [64]byte  // Fixed-size array вҶ’ stack-allocated
    binary.LittleEndian.PutUint32(buf[:], magic)
    return buf[:]  // вҡ  Escapes! Compiler moves to heap.
}

// STACK: stays local, never escapes
func readMagic(r io.Reader) (uint32, error) {
    var buf [4]byte  // 4 bytes on the stack
    _, err := io.ReadFull(r, buf[:])
    return binary.LittleEndian.Uint32(buf[:]), err
}

// HEAP: make() with runtime-determined size вҶ’ always heap
func readEntry(size int) []byte {
    data := make([]byte, size)  // Heap allocation
    return data
}
Performance Rule

Use var buf [N]byte (fixed array) for small, known-size buffers. Use make([]byte, n) for dynamic sizes. The compiler's escape analysis is your friend — check it with go build -gcflags='-m'.

Contiguous vs Scattered Memory

Contiguous
All bytes adjacent in memory. CPU cache loves this. Arrays, slices, mmap regions. One pointer + length = full access.
Scattered
Bytes spread across heap. Linked lists, maps of slices, interface boxing. Cache misses on every hop.

The Trap

Sub-slices hold the entire backing array alive. If you read a 100MB file into buf, then keep name := buf[0:10], the GC cannot free the other 99.99MB. Fix: name := append([]byte{}, buf[0:10]...) to copy.

Pre-allocation: The make([]byte, 0, cap) Pattern

// BAD: grows 10 times, each time copies everything
var buf []byte
for i := 0; i < 10; i++ {
    buf = append(buf, chunk...)  // May reallocate each time
}

// GOOD: one allocation, ten appends, zero copies
buf := make([]byte, 0, estimatedSize)
for i := 0; i < 10; i++ {
    buf = append(buf, chunk...)  // Uses existing capacity
}

// BEST for known-size: direct indexing, no append overhead
buf := make([]byte, totalSize)
copy(buf[0:], header)
copy(buf[headerLen:], payload)

The Reset() Pattern

buf.Reset() keeps the allocated memory but zeroes the logical length. This means you can reuse the same buffer across iterations without allocating. Your shard tests could benefit: instead of var buf bytes.Buffer in each subtest, declare once and buf.Reset() between iterations.

bytes.Buffer vs bytes.Reader

bytes.Buffer

Read and write. Grows dynamically. Good for building data incrementally.

var buf bytes.Buffer
buf.Write(header)
buf.Write(payload)
// Now read back
h, _ := ReadShardHeader(&buf)

bytes.Reader

Read-only. Supports io.Seeker. No allocations. Better for parsing existing data.

r := bytes.NewReader(data)
r.Seek(64, io.SeekStart) // Jump!
// Implements io.ReaderAt too
r.ReadAt(buf, offset)
When To Use Which

bytes.Buffer — building data (encoder output, test roundtrips). bytes.Reader — parsing existing data (decoder input, random access). []byte directly — known-size, performance-critical (header encoding).

4. Binary Encoding

There are two ways to serialize structs to bytes in Go: the safe way (binary.Write) and the fast way (direct LittleEndian.Put* into slices). Your shard codebase chose the fast way — and it's 2-3x faster because it avoids reflection entirely.

The Slow Path: binary.Write

// binary.Write uses reflection to walk struct fields
// ~180ns per call, allocates temporary buffer internally
type Header struct {
    Magic   uint32
    Version uint16
    Flags   uint16
}
h := Header{Magic: 0xSHRD, Version: 2, Flags: 0}
err := binary.Write(w, binary.LittleEndian, &h)  // Reflection!
slow path

The Fast Path: Direct Put

// Direct encoding: ~60ns, zero allocations beyond the buffer
buf := make([]byte, 8)
binary.LittleEndian.PutUint32(buf[0:4], 0xSHRD)  // 4 bytes at offset 0
binary.LittleEndian.PutUint16(buf[4:6], 2)       // 2 bytes at offset 4
binary.LittleEndian.PutUint16(buf[6:8], 0)       // 2 bytes at offset 6
w.Write(buf)
fast path вҖ” your shard code does this

func EncodeTensorV1(t *TensorV1) ([]byte, error) {
    headerSize := 4 + len(t.Dims)*8
    total := headerSize + len(t.Data)
    buf := make([]byte, total)  // Single allocation for everything

    // Pack header directly into buffer
    buf[0] = byte(t.DType)              // 1 byte
    buf[1] = byte(len(t.Dims))           // 1 byte
    binary.LittleEndian.PutUint16(buf[2:4], 0)  // flags reserved

    for i, d := range t.Dims {
        off := 4 + i*8
        binary.LittleEndian.PutUint64(buf[off:off+8], d)
    }

    copy(buf[headerSize:], t.Data)     // Bulk copy raw tensor data
    return buf, nil
}

6. Buffer Pooling with sync.Pool

When you're encoding thousands of tensors in a hot loop, allocating and freeing buffers becomes the bottleneck — not the encoding itself. sync.Pool is Go's answer: a concurrent-safe free list that the GC can drain when memory is tight.

var bufferPool = sync.Pool{
    New: func() any {
        b := make([]byte, 0, 4096)  // 4KB default cap
        return &b
    },
}

func EncodeTensorPooled(t *TensorV1) ([]byte, error) {
    bp := bufferPool.Get().(*[]byte)
    buf := (*bp)[:0]  // Reset length, keep capacity

    // Encode into pooled buffer
    buf = appendTensorHeader(buf, t)
    buf = append(buf, t.Data...)

    // Copy result out (caller owns this)
    result := make([]byte, len(buf))
    copy(result, buf)

    // Return buffer to pool
    *bp = buf
    bufferPool.Put(bp)

    return result, nil
}

Pool Pitfalls

1. Never store pointers into pooled buffers — they'll be reused by other goroutines. 2. The GC can drain the pool at any time — don't rely on it for caching. 3. 1. Alignment must be correct (float32 needs 4-byte alignment — your 64-byte shard alignment guarantees this). 2. Endianness must match (your format is LE, x86 is LE — works). 3. With mmap, this is fine because you control the lifetime via Close().

// A simple ring buffer for streaming tensor chunks
type Ring struct {
    buf  []byte
    size int
    r, w int  // read and write cursors
    full bool
}

func NewRing(size int) *Ring {
    return &Ring{buf: make([]byte, size), size: size}
}

func (r *Ring) Write(p []byte) (int, error) {
    n := 0
    for _, b := range p {
        if r.full {
            return n, io.ErrShortWrite
        }
        r.buf[r.w] = b
        r.w = (r.w + 1) % r.size  // Wrap around!
        r.full = r.w == r.r
        n++
    }
    return n, nil
}

func (r *Ring) Read(p []byte) (int, error) {
    n := 0
    for i := range p {
        if r.r == r.w && !r.full {
            if n == 0 { return 0, io.EOF }
            return n, nil
        }
        p[i] = r.buf[r.r]
        r.r = (r.r + 1) % r.size  // Wrap around!
        r.full = false
        n++
    }
    return n, nil
}

Lock-Free Ring Buffer (SPSC)

For single-producer single-consumer scenarios (like feeding tensor data to a GPU), you can avoid locks entirely using atomic operations:

type SPSCRing struct {
    buf  []byte
    cap  uint64
    _pad1 [56]byte  // Prevent false sharing (64B cache line)
    head atomic.Uint64  // Writer increments
    _pad2 [56]byte
    tail atomic.Uint64  // Reader increments
}

// Writer (producer goroutine)
func (r *SPSCRing) TryWrite(b byte) bool {
    head := r.head.Load()
    next := (head + 1) % r.cap
    if next == r.tail.Load() {
        return false  // Full
    }
    r.buf[head] = b
    r.head.Store(next)  // Release semantics
    return true
}
Where Ring Buffers Shine

Streaming decompression: zstd decompressor writes into ring, consumer reads tensors out. Fixed memory regardless of file size. GPU feeding: CPU fills ring with weight data, GPU DMA reads from the other end. WAL buffers: your DeepData WAL could use a ring for the in-memory write buffer before flushing to disk.

type Arena struct {
    buf []byte
    pos int
}

func NewArena(size int) *Arena {
    return &Arena{buf: make([]byte, size)}
}

// Alloc returns a sub-slice. O(1), no GC pressure.
func (a *Arena) Alloc(n int) []byte {
    if a.pos+n > len(a.buf) {
        panic("arena exhausted")
    }
    s := a.buf[a.pos : a.pos+n]
    a.pos += n
    return s
}

// AllocAligned returns aligned memory
func (a *Arena) AllocAligned(n, align int) []byte {
    aligned := (a.pos + align - 1) &^ (align - 1)
    if aligned+n > len(a.buf) {
        panic("arena exhausted")
    }
    s := a.buf[aligned : aligned+n]
    a.pos = aligned + n
    return s
}

// Reset frees everything at once. Zero GC cost.
func (a *Arena) Reset() { a.pos = 0 }
